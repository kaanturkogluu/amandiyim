Genel Bakış

Amandiyim, modern ve güvenli bir web uygulaması geliştirmek için tasarlanmış bir PHP framework'üdür.

Özellikler

  • MVC (Model-View-Controller) mimarisi
  • Güvenli form işlemleri
  • Veritabanı yönetimi
  • Oturum yönetimi
  • CSRF koruması
  • XSS koruması

Kurulum

Projeyi yerel ortamınıza kurmak için aşağıdaki adımları takip edin:

git clone https://github.com/kullanici/amandiyim.git
cd amandiyim
composer install
cp .env.example .env
php -S localhost:3000
Not: Composer'ın sisteminizde kurulu olduğundan emin olun.

Veritabanı İşlemleri

Veritabanı işlemleri için Database sınıfını kullanabilirsiniz:

$db = Database::getInstance();

// Veri ekleme
$db->insert('users', [
  'name' => 'John Doe',
  'email' => 'john@example.com'
]);

// Veri sorgulama
$user = $db->fetch("SELECT * FROM users WHERE id = ?", [1]);

Form İşlemleri

Form işlemleri için Form sınıfını kullanabilirsiniz:

$form = new Form();

$form->addField('name', 'text', [
  'required' => true,
  'class' => 'form-control'
]);

echo $form->render();
Önemli: Form işlemlerinde CSRF token'ı otomatik olarak eklenir.

Güvenlik

Framework'ün güvenlik özellikleri:

CSRF Koruması

// Token oluşturma
$token = Csrf::generateToken();

// Token doğrulama
if (Csrf::validateToken($_POST['_token'])) {
  // İşlem güvenli
}

XSS Koruması

// Veriyi güvenli hale getirme
$safeData = Helper::escape($userInput);

Yardımcı Sınıflar

Framework'de kullanılan yardımcı sınıflar:

Helper Sınıfı

// URL oluşturma
$url = Helper::router('login.php');

// Güvenli çıktı
echo Helper::escape($data);

// Slug oluşturma
$slug = Helper::slug('Merhaba Dünya');

Hata Ayıklama

Hata ayıklama için debug modunu kullanabilirsiniz:

// Debug modunu aktifleştirme
$db->debug = true;

// Hata mesajlarını görüntüleme
if (Helper::isDebug()) {
  echo $error->getMessage();
}
Not: Production ortamında debug modunu kapatmayı unutmayın.

Session İşlemleri

Session yönetimi için Session sınıfını kullanabilirsiniz:

Session Başlatma

// Session instance'ını al
$session = Session::getInstance();

// Session otomatik olarak başlatılır
// Güvenli session ayarları yapılandırılır

Temel Session İşlemleri

// Veri ekleme
$session->set('user_id', 1);

// Veri alma
$userId = $session->get('user_id');

// Veri silme
$session->delete('user_id');

// Veri kontrolü
if ($session->has('user_id')) {
  // Veri mevcut
}

Flash Mesajları

// Flash mesajı ekleme
$session->flash('success', 'İşlem başarılı!');

// Flash mesajı alma
$success = $session->getFlash('success');

// Tüm flash mesajlarını alma
$allFlash = $session->getAllFlash();

Form Verileri

// Form verilerini saklama
$session->storeOld($_POST);

// Eski form verilerini alma
$email = $session->old('email');

// Tüm eski verileri alma
$allOld = $session->getAllOld();

Hata ve Başarı Mesajları

// Hata mesajı ekleme
$session->addError('Geçersiz şifre');

// Hata mesajlarını alma
$errors = $session->getErrors();

// Başarı mesajı ekleme
$session->setSuccess('Kayıt başarılı!');

// Başarı mesajını alma
$success = $session->getSuccess();

Session Güvenliği

// Session ID'yi yenileme
$session->regenerate();

// Session'ı temizleme
$session->clear();

// Session süresini kontrol etme
if ($session->isExpired(1800)) {
  // Session süresi dolmuş
}
Not: Session sınıfı singleton pattern kullanır ve otomatik olarak güvenli session ayarlarını yapılandırır.
Önemli: Session verilerini hassas bilgilerle doldurmaktan kaçının ve session süresini uygun şekilde ayarlayın.

Form Veri İşleme

Form verilerini işlemek için birkaç yöntem kullanabilirsiniz:

POST Verilerini Alma

// POST verilerini doğrudan alma
$email = $_POST['email'] ?? null;
$password = $_POST['password'] ?? null;

// Tüm POST verilerini alma
$data = $_POST;

// Belirli bir veriyi kontrol ederek alma
if (isset($_POST['email']) && !empty($_POST['email'])) {
  $email = $_POST['email'];
} else {
  $errors[] = 'E-posta adresi gereklidir.';
}

Form Sınıfı ile Veri Alma

// Form sınıfı oluşturma
$form = new Form();

// Form alanlarını tanımlama
$form->addField('email', 'email', [
  'required' => true,
  'label' => 'E-posta Adresi'
]);

$form->addField('password', 'password', [
  'required' => true,
  'label' => 'Şifre'
]);

// Form gönderildiğinde
if ($_SERVER['REQUEST_METHOD'] === 'POST') {
  $data = $form->getData();
  $errors = $form->getErrors();

  if (empty($errors)) {
    // Form verileri geçerli
    $email = $data['email'];
    $password = $data['password'];
  }
}

Form Verilerini Doğrulama

// Form verilerini doğrulama
$validator = new Validator();

$validator->setRules([
  'email' => 'required|email',
  'password' => 'required|min:6'
]);

if ($validator->validate($_POST)) {
  $data = $validator->getData();
} else {
  $errors = $validator->getErrors();
}

Form Verilerini Session'da Saklama

// Form verilerini session'da saklama
$session = Session::getInstance();

if ($_SERVER['REQUEST_METHOD'] === 'POST') {
  if (empty($errors)) {
    // Veriler geçerli, işleme devam et
    $session->set('user_email', $_POST['email']);
  } else {
    // Hatalar varsa form verilerini sakla
    $session->storeOld($_POST);
    $session->addError('Lütfen formu düzeltin.');
  }
}

// Eski form verilerini alma
$email = $session->old('email');
$password = $session->old('password');
Not: Form verilerini işlerken her zaman güvenlik kontrollerini yapın ve XSS koruması uygulayın.
Önemli: Hassas verileri (şifre gibi) session'da saklamaktan kaçının ve her zaman güvenli bir şekilde işleyin.

Güvenli POST Veri İşleme

POST verilerini güvenli bir şekilde işlemek için aşağıdaki yöntemleri kullanabilirsiniz:

Helper Sınıfı ile Güvenli Veri Alma

// POST verilerini güvenli bir şekilde alma
$email = Helper::post('email');
$password = Helper::post('password');

// Varsayılan değer ile alma
$name = Helper::post('name', 'Misafir');

// Belirli bir veriyi kontrol ederek alma
if (Helper::hasPost('email')) {
  $email = Helper::post('email');
} else {
  $errors[] = 'E-posta adresi gereklidir.';
}

XSS Koruması ile Veri Alma

// XSS korumalı veri alma
$safeData = Helper::escape($_POST['user_input']);

// HTML içeriğini güvenli hale getirme
$safeHtml = Helper::sanitizeHtml($_POST['html_content']);

// URL'yi güvenli hale getirme
$safeUrl = Helper::sanitizeUrl($_POST['url']);

Form Sınıfı ile Güvenli Veri İşleme

// Form sınıfı oluşturma
$form = new Form();

// Form alanlarını tanımlama
$form->addField('email', 'email', [
  'required' => true,
  'sanitize' => true,
  'validate' => 'email'
]);

// Form verilerini güvenli bir şekilde alma
if ($_SERVER['REQUEST_METHOD'] === 'POST') {
  $data = $form->getData();
  $errors = $form->getErrors();

  if (empty($errors)) {
    // Veriler güvenli ve geçerli
    $email = $data['email'];
  }
}

Validator Sınıfı ile Veri Doğrulama

// Validator sınıfı oluşturma
$validator = new Validator();

// Doğrulama kuralları tanımlama
$validator->setRules([
  'email' => 'required|email|max:255',
  'password' => 'required|min:6|max:20',
  'username' => 'required|alpha_num|min:3|max:20'
]);

// Verileri doğrulama
if ($validator->validate($_POST)) {
  $data = $validator->getData();
} else {
  $errors = $validator->getErrors();
}
Not: Tüm kullanıcı girdilerini işlerken her zaman güvenlik kontrollerini yapın ve XSS koruması uygulayın.
Önemli: Hassas verileri (şifre gibi) her zaman güvenli bir şekilde işleyin ve asla düz metin olarak saklamayın.

Login İşlemleri

Kullanıcı girişi ve oturum yönetimi için Login sınıfını kullanabilirsiniz:

Login Sınıfını Başlatma

// Login sınıfını başlatma
$login = Login::getInstance();

Kullanıcı Girişi

// POST verilerini güvenli bir şekilde alma
$email = Helper::post('email');
$password = Helper::post('password');

// Giriş işlemi
if ($login->login($email, $password)) {
  // Başarılı giriş
  header('Location: /dashboard');
  exit;
} else {
  // Hata mesajlarını al
  $errors = $login->getErrors();
}

Oturum Kontrolü

// Kullanıcı giriş yapmış mı kontrol etme
if ($login->isLoggedIn()) {
  // Kullanıcı bilgilerini al
  $user = $login->getUser();
  echo "Hoş geldiniz, " . $user['name'];
} else {
  // Giriş yapılmamış
  header('Location: /login');
  exit;
}

Çıkış Yapma

// Çıkış yapma
$login->logout();

// Ana sayfaya yönlendirme
header('Location: /');
exit;

Güvenlik Özellikleri

  • XSS koruması
  • SQL Injection koruması
  • Brute force koruması (5 başarısız denemeden sonra 15 dakika kilit)
  • Session güvenliği
  • Şifre doğrulama (password_verify)

Veri Doğrulama

  • E-posta formatı kontrolü
  • Şifre uzunluğu kontrolü
  • Boş alan kontrolü
  • Hesap aktiflik kontrolü
Not: Login sınıfı singleton pattern kullanır, bu yüzden her zaman getInstance() metodu ile başlatılmalıdır.
Önemli: Hassas verileri (şifre gibi) her zaman güvenli bir şekilde işleyin ve asla düz metin olarak saklamayın.

Blueprint

Blueprint sınıfı, veritabanı tablolarını oluşturmak için kullanılan bir yapı sağlar. Bu sınıf, tablo oluşturma işlemlerini daha kolay ve okunabilir hale getirir.

Kullanım


$blueprint = new Blueprint('users');
$blueprint->id()
          ->string('name')
          ->string('email')
          ->timestamps();

Mevcut Metodlar

  • id() - Otomatik artan birincil anahtar kolonu ekler
  • string($name, $length = 255) - VARCHAR tipinde kolon ekler
  • text($name) - TEXT tipinde kolon ekler
  • integer($name) - INT tipinde kolon ekler
  • decimal($name, $precision = 10, $scale = 2) - DECIMAL tipinde kolon ekler
  • enum($name, $values) - ENUM tipinde kolon ekler
  • boolean($name) - BOOLEAN tipinde kolon ekler
  • timestamp($name) - TIMESTAMP tipinde kolon ekler
  • foreignKey($column, $referenceTable, $referenceColumn = 'id') - Foreign key ekler
  • nullable() - Son eklenen kolonu NULL olabilir yapar
  • default($value) - Son eklenen kolona varsayılan değer atar
  • unique() - Son eklenen kolonu benzersiz yapar
  • timestamps($value = true) - created_at ve updated_at kolonlarını ekler
  • primaryKey($columns) - Birincil anahtar kolonlarını belirler

Örnek Kullanım


$blueprint = new Blueprint('products');
$blueprint->id()
          ->string('name')
          ->text('description')
          ->decimal('price', 10, 2)
          ->boolean('is_active')
          ->foreignKey('category_id', 'categories')
          ->timestamps();

Oluşturulan SQL

CREATE TABLE IF NOT EXISTS `products` ( `id` INT AUTO_INCREMENT NOT NULL, `name` VARCHAR(255) NOT NULL, `description` TEXT NOT NULL, `price` DECIMAL(10, 2) NOT NULL, `is_active` BOOLEAN NOT NULL, `category_id` INT NOT NULL, `created_at` TIMESTAMP DEFAULT CURRENT_TIMESTAMP, `updated_at` TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP, PRIMARY KEY (`id`), FOREIGN KEY (`category_id`) REFERENCES `categories`(`id`) ON DELETE CASCADE ) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci;
